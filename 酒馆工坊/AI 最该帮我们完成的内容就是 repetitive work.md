## 我用 Claude Code 做了个 Markdown → 公众号排版工具

### 问题：公众号排版很烦

最近自己在发布公众号前，我就觉得公众号调格式很麻烦，早些时候的几篇就主打一个能凑合看就行

但是后面我就 vibecoding 了一个 markdown 转 html 的文件，让我一键可以使用。

简单来说就是我习惯用 Markdown 写东西了。Obsidian也一直是我的写作工具，但是微信公众号展示的并不是 markdown ，所以复制到公众号后需要自己调整成期望展示成的样子。

手动调？太费时间。每次发文章都要重新调一遍，典型的 repetitive work，虽然公众号也有模板，但是我也不知道同一个模板是否会看腻，我也不知道未来我看腻后，会不会因为想要再调整而增加心智负担因此不发。
（我的小红书就因为我懒得拼图还没有跟上我公众号的更新速度），希望我未来可以填下这个高效发小红书的坑

---

### 现有工具的问题

我也有试过一些现成的 Markdown 转换工具：

- 有的需要登录，样式太花哨
- 有的配置太复杂，学习成本高
- 有的要付费

现在新的软件都是层出不穷的，我总认为一个软件的学习成本也很高，折腾到最后也未必是好用的
作为新手，我也不知道段落间距、字号该设多少。

与其到处找微信公众号的文章当做模板参考，不如自己直接指挥 AI，生成一个大概能看的就行了。

---

### 解决方案

现在很多琐碎的事我都喜欢扔给 AI，尤其是写一些mardown 也可以调整格式的时候我也扔给 AI，

因此我就让 Claude Code 帮我写了个 Markdown → HTML 转换工具，这样我就可以一键复制一下我所需要的样式，而不是去后台编辑器里反复操作了

由于我自己的个人网站里本身就已经有做 markdown 展示，因此开发成本也比较低

**核心功能：**

1. 左侧粘贴 Markdown，右侧实时预览 HTML
2. 几套预设主题可以切换
3. 一键复制，直接粘贴到公众号后台

### 一些注意事项

**Markdown 写作建议：**
- 列表不要嵌套太深（微信支持有限）
- 代码块建议 ≤ 20 行
- 图片需要单独上传
- 粘贴后检查一下链接和加粗

---

### AI 应该做 repetitive work

这个工具让我意识到：AI 最该帮我们做的，就是重复性、机械性的工作——比如格式转换、样式调整。

但写作的核心（构思、表达、情感）还是得自己来。工具只是扫清障碍，让我能把精力放在真正重要的事上。
