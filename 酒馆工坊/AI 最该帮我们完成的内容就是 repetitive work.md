## 我用 Claude Code 做了个 Markdown → 公众号排版工具

### 痛点：Markdown 到公众号的距离

最近发布公众号前，发现在微信公众号后台调整文字格式很麻烦。

我习惯用 Markdown 写东西了，Obsidian 也一直是我的写作工具，但是微信公众号不支持 Markdown 格式，所以复制到公众号后需要自己调整成期望展示的样子。

但是当时也不力求做到完美，第一篇就主打一个能凑合看就行。

在网上我也试过一些现成的 Markdown 转换工具：
基本都需要登录，也有些很成熟的框架需要付费，有的配置太复杂，学习成本高。

但是现在新的软件都是层出不穷的，我总认为每个软件的学习成本都很高，
况且作为新手，我也不知道段落间距、字号该设多少。

主要是学习成本太高，我怕折腾到最后未必好用。

手动调？太费时间。每次发文章都要重新调一遍，典型的 repetitive work，虽然微信也有模板功能，但初始化模板依旧要靠自己，与其到处找微信公众号的文章当做模板参考，不如直接指挥 AI，生成一个看得过去的版本就行。

而且我不知道同一个模板是否会看腻。也不知道未来看腻后，会不会因为想要再调整而增加心智负担因此不发。

因此我想干脆写一个 Markdown 转 HTML 的界面，让我一键可以使用。

（我的小红书就因为我懒得拼图还没跟上我公众号的更新速度），希望我未来可以填上这个高效发小红书的坑。

---

### 解决方案

现在很多琐碎的事我都喜欢扔给 AI，尤其是需要调整 Markdown 格式的时候。

因此我就让 Claude Code 帮我写了个 Markdown → HTML 转换工具，这样我就可以一键复制我所需要的样式，而不是去后台编辑器里反复操作了。

由于我的个人网站本身就有 Markdown 展示功能，因此开发成本也比较低。

**核心功能：**
1. 左侧粘贴 Markdown，右侧实时预览 HTML
2. 几套预设主题可以切换
3. 一键复制，直接粘贴到公众号后台

### 一些注意事项

**Markdown 写作建议：**
- 列表不要嵌套太深（微信支持有限）
- 代码块建议 ≤ 20 行
- 图片需要单独上传
- 粘贴后检查一下链接和加粗

---

### AI 应该做 repetitive work

这个工具让我意识到：AI 最该帮我们做的，就是重复性、机械性的工作——比如格式转换、样式调整。

但写作的核心（构思、表达、情感）还是得自己来。工具只是扫清障碍，让我能把精力放在真正重要的事上。
