## Claude真的会让人降智
今天又在折腾 claudecode 了
我发现只要自己在动脑子思考，就会有很多事情感觉做不完，
聊一下最近 claudecode 的另一个感受

claudecode有时候给到的计划不一定会完全看仔细，但这其实是不好的行为
因为有的时候 claudecode 会提出一些改进方案，在终端里阅读方案其实并没有很方便，有的时候精神状况不是很好的时候，看到 claude的界面提问 Do you want to proceed?只想敲一个 yes，可能是我的终端样式配置不够友好，我觉得他提出的方案应该高亮才对

或者像现在很多软件的用户协议，强制用户滚动到底且阅读到一定时长才允许点击"同意"

狂按回车真的很爽，但这个动作可能就和划手机一样是"无意识"，"不需要学习"，"不费脑"的动作
有的时候可能脑子都不动直接敲 enter 键了，等"啪"敲完再"啪一拍脑袋"：哎呀方案又没看清楚。

等出现 bug 后，发现越改问题越多时，就会需要去复盘原有的计划是否出问题了

但是这时候的聊天记录可能已经非常长了，在终端里会看聊天记录的交互也非常差，因此我觉得可以去定期导出日志，便于在有需要的时候复盘

## 具体思路
相信也有很多人可能和我一样，可能在多个仓库里同时让 claude 打工
因此我对它的要求就是：
- 基于日期输出一份 markdown
- 基于项目也输出一份 markdown

基于日期的 markdown：便于一日/周复盘，这周具体做了什么事。
基于项目的 markdown： 便于自己/组员定位问题

## 实现流程
依旧是基于 claudecode 的 hook，当一个任务完成时，去总结输出刚刚的 task 具体需求以及实现逻辑、以及改动的内容。

需要每日复盘自己一天做了什么，就和学生时代需要一天去总结巩固一下白天学习的知识，巩固加强的过程才是最重要的，不要过度高估自己的"大脑"。 巩固过后再次输出的内容才是属于自己的知识财富
