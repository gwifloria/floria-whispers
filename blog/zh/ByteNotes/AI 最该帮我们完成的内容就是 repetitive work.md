### 痛点：Markdown 到公众号的距离

上周在公众号后台调了半小时格式，突然意识到：这种重复劳动为什么不能自动化？

我习惯用 Markdown 写东西，Obsidian 也一直是我的写作工具。但微信公众号不支持 Markdown 格式，每次复制过去都需要手动调整排版。

**试过现成工具，但都不太理想：**
- 需要登录，或者付费
- 配置复杂，学习成本高
- 作为新手，我也不知道段落间距、字号该设多少

**手动调？更麻烦：**
- 每次发文都要重新调一遍，典型的 repetitive work
- 微信虽然有模板功能，但初始化模板还是要靠自己
- 而且我也担心：同一个模板看腻了怎么办？想调整会不会又增加心智负担，导致我干脆不想发了？

与其到处找文章当模板参考，不如让 AI 直接生成一个看得过去的版本。

**于是我花了一个下午，让 Claude Code 帮我写了个 Markdown 转 HTML 的界面，一键复制就能用。**

> 题外话：我的小红书就因为懒得拼图，还没跟上公众号的更新速度。希望未来能填上这个"高效发小红书"的坑。

---

### 思路：让 AI 处理格式转换

现在很多琐碎的事我都喜欢扔给 AI，尤其是需要调整 Markdown 格式的时候。

因此我就让 Claude Code 帮我写了个 Markdown → HTML 转换工具，这样我就可以一键复制我所需要的样式，而不是去后台编辑器里反复操作了。

由于我的个人网站本身就有 Markdown 展示功能，因此开发成本也比较低。

**核心功能：**
1. 左侧粘贴 Markdown，右侧实时预览 HTML
2. 几套预设主题可以切换
3. 一键复制，直接粘贴到公众号后台

### 一些注意事项

**Markdown 写作建议：**
- 列表不要嵌套太深（微信支持有限）
- 代码块建议 ≤ 20 行
- 图片需要单独上传
- 粘贴后检查一下链接和加粗

### 工具的意义：把时间花在创作上

这个工具让我意识到：AI 最该帮我们做的，就是重复性、机械性的工作——比如格式转换、样式调整。

但写作的核心（构思、表达、情感）还是得自己来。工具只是扫清障碍，让我能把精力放在真正重要的事上。
