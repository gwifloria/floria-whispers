AI 博主分类系统 - Phase 1 实现计划

 目标

 添加 "AI" 分类，支持筛选和搜索 AI 博主，为后续内容聚合功能打基础。

 当前状态分析
 ┌─────────────────────┬───────────────────────────────┬────────────────┐
 │        模块         │             现状              │      问题      │
 ├─────────────────────┼───────────────────────────────┼────────────────┤
 │ 前端 FilterPanel    │ 6 个硬编码分类                │ 缺少 AI 分类   │
 ├─────────────────────┼───────────────────────────────┼────────────────┤
 │ 爬虫 xiaohongshu.py │ _guess_category() 有 8 个分类 │ 缺少 AI 关键词 │
 ├─────────────────────┼───────────────────────────────┼────────────────┤
 │ API                 │ 支持 ?category=xxx            │ 无需修改       │
 ├─────────────────────┼───────────────────────────────┼────────────────┤
 │ 数据模型            │ category: string              │ 无类型约束     │
 └─────────────────────┴───────────────────────────────┴────────────────┘
 核心问题: 分类定义分散在前端和爬虫中，不统一。

 ---
 实现方案

 设计原则

 1. 集中管理分类配置 - 在 api.ts 创建 CATEGORY_CONFIG
 2. 类型安全 - 使用 CategoryKey 类型约束
 3. 可扩展 - 便于未来添加新分类
 4. 向后兼容 - 现有数据通过迁移脚本更新
