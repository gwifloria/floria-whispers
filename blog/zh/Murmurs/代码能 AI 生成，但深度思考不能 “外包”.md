
#### 01 Claude Code 的双面性：便利与 bug 并存

最近 Claude Code 有点让我烦，我发现用它来解决一些简单的任务还挺方便，但它又总是会出现一些 bug，毕竟人和机器的沟通还是会存在信息差。

我发现它在改 bug 的时候不断造出一些新的 bug，我开始怀疑：Claude Code 真的提升了我的生产力吗？

人都喜欢走捷径偷懒，好像这是大家的通病。只要一件事看起来能很轻松搞定，我们就放松警惕。

#### 02 便捷变成陷阱

有些人可能不了解：Claude Code 有个「Plan Mode」，当你提交一个 prompt 的时候，它会在执行前告诉你详细的计划，让你决定是否采用这个方案，避免了 AI 一顿输出后，才发现与自己预期不符的情况。

但是在我很累的时候，我发现我会直接按下回车，事后才突然意识到还没仔细看计划 —— 按回车这件事太简单了。

当某件事不再需要任何努力时，我们的专注力就消失了。我们不再是主动选择，而只是机械地点击。注意力开始飘移 —— 飘向下一件事、下一个通知、下一个多巴胺刺激。

#### 03 问题不在工具，而在注意力

这其实并不是在说 Claude Code，而是在说注意力 —— 以及我们是如何一点点丧失这项技能的。

我最近在读《刻意练习》和《手机断舍离》。这两本书都在反复强调一个道理：深度专注。

“要记住：如果你在走神，或者你很放松，并且只为了好玩，你可能不会进步。”

所以当我们谈论「AI 提升生产力」时，我们往往自己思考得更少，却要求工具干得更多。久而久之，这种习惯会让我们越来越不愿意去做那些需要专注和深度思考的事。

如果我们一直这样被分心、不愿放慢脚步、不愿「浪费时间」去真正吸收一些东西 —— 多年之后，我们还会具备深度思考、甚至基本独立思考的能力吗？

#### 04 别被工具牵着走

Claude Code 终究只是一个工具。我们应该学会尽可能减少重复劳动，而不是沉溺于那种「看似在做事、其实什么都不用投入」的幻觉。
