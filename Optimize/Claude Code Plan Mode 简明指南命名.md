
## 🚀 最简单的开始方式

### 第一步：创建计划文件

```bash
touch plan.md
```

### 第二步：在 plan.md 里写清楚你要做什么

```markdown
# 我要做什么
重构博客项目的数据获取层

# 现在的问题
- 服务端和客户端用不同的方式获取数据
- 代码重复，难以维护
- 错误处理不统一

# 我想要的结果
- 统一的数据获取接口
- 更好的性能
- 清晰的代码结构

# 具体要做的事
1. 创建统一的数据服务层
2. 重构现有组件
3. 优化API路由
4. 测试所有功能

# 不能破坏的东西
- 现有功能必须正常工作
- SEO 不能受影响
- 性能不能变差
```

### 第三步：启动 Plan Mode

```
/plan @plan.md

开始执行，有问题就问我
```

## 📝 计划文件的必需内容

### 🎯 四个核心部分（必须有）

1. **要做什么** - 一句话描述目标
2. **现在的问题** - 为什么要做这件事
3. **想要的结果** - 做完后应该是什么样
4. **具体步骤** - 要做哪些事情

### ✨ 可选的增强内容

- **技术约束** - 不能用什么/必须用什么
- **时间要求** - 什么时候完成
- **风险提醒** - 什么地方容易出错
- **验收标准** - 怎么判断做完了

## 💡 实际使用技巧

### 对话框里只说这些：

- ✅ "开始执行"
- ✅ "继续下一步"
- ✅ "这里有问题，先停"
- ✅ "改一下计划：[具体变化]"

### 不要在对话框里：

- ❌ 重复写一遍完整需求
- ❌ 粘贴大段代码
- ❌ 解释项目背景

## 🔧 完整示例

### plan.md 示例

```markdown
# 添加用户认证功能

## 要做什么
给博客添加登录注册功能

## 现在的问题  
- 没有用户系统
- 无法区分访客和管理员
- 无法保护敏感页面

## 想要的结果
- 用户可以注册登录
- 管理员可以发布文章
- 敏感页面需要登录才能访问

## 具体步骤
1. 集成 NextAuth.js
2. 添加登录注册页面
3. 保护管理页面
4. 测试登录流程

## 技术要求
- 使用 NextAuth.js
- 支持 GitHub OAuth
- 保持现有页面不变
```

### 使用流程

```
1. 写好 plan.md
2. 运行：/plan @plan.md
3. 对话：开始执行
4. Claude 按步骤做，有问题会问你
5. 你简短回答或调整
```
